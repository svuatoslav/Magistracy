# Magistracy – Satellite Orbit Simulation in Unity

Проект магистерской работы по моделированию движения искусственного спутника вокруг планеты с использованием численных методов и визуализации в Unity.  
Основная логика моделирования и кода расположена в папке `Assets/Scripts`.

## Основные возможности

- Численное интегрирование уравнений движения спутника в орбитальной задаче (ODE).
- Реализация и использование методов Рунге–Кутты с таблицами Бутчера для интегрирования.
- Задание орбитальных параметров и конфигураций через конфигурационные файлы (`Configs`).
- Визуализация орбиты спутника вокруг планеты в Unity-сцене с использованием готовых ассетов (планета, небо, UI).
- Отображение осей, планеты и траектории спутника, управление камерой.

## Структура проекта

Ключевые папки с написанным кодом:

- `Assets/Scripts/RungeKutta` – реализации методов Рунге–Кутты и связанные структуры (включая таблицы Бутчера и обработку ошибок).
- `Assets/Scripts/Kepler` – код, связанный с кеплеровскими элементами орбиты и их использованием в симуляции.
- `Assets/Scripts/Orbits` – логика работы с орбитами, построения и обновления орбитального движения.
- `Assets/Scripts/Structs` – структуры данных для представления углов Эйлера и безразмерных импульсов спутника.
- `Assets/Scripts/Configs` – конфигурации и интерфейсы для применения конфигов.
- `Assets/Scripts/DataSerialiizers` – загрузка и сериализация данных/конфигураций для симуляции.
- `Assets/Scripts/Errors` – расчёт и хранение ошибок интегрирования/аппроксимации.
- `Assets/Scripts/UI` – элементы пользовательского интерфейса для управления симуляцией и отображения данных.
- Отдельные скрипты: `Planet.cs`, `Satellite.cs`, `IODEMotions.cs` – компоненты для планеты, спутника и численной интеграции уравнений движения.
- `CameraController.cs`, `Axes.cs` – управление камерой и отрисовка осей.

## Используемые технологии

- Unity – 3D сцена, визуализация планеты, спутника, UI и управление камерой.
- C# – реализация численных методов, структур данных и логики симуляции.
- Методы Рунге–Кутты разных порядков с таблицами Бутчера (в том числе с оценкой ошибки).
- Кеплеровские элементы орбиты и базовая орбитальная механика для задания начальных условий.

## Фокус работы

В рамках магистерской работы основной акцент сделан на:

- Реализацию численных методов интегрирования (семейство Runge–Кутты) и инфраструктуры под них.
- Организацию кода для настройки и запуска орбитальной симуляции через конфиги и сериализацию данных.
- Интеграцию моделирования с Unity для наглядной визуализации результатов.

Все скрипты в `Assets/Scripts` написаны и/или модифицированы в рамках этой работы; остальные ассеты используются как готовые библиотеки для визуализации и окружения.
